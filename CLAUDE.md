# CLAUDE.md

이 파일은 Claude Code(claude.ai/code)가 이 저장소의 코드로 작업할 때 참고하는 가이드입니다.

## 프로젝트 개요
HTML, CSS, JavaScript, TailwindCSS를 사용하여 개발자 웹 이력서를 구축하는 프로젝트입니다.

## 기술 스택
- **HTML5** - 구조 및 마크업
- **CSS3 / TailwindCSS** - 스타일링 및 레이아웃
- **JavaScript (Vanilla)** - 상호작용 및 동작
- **Git** - 버전 관리

## 언어 및 커뮤니케이션 규칙

### 응답 언어
- **기본 응답 언어**: 한국어
- 모든 설명, 제안, 피드백은 한국어로 작성

### 코드 작성 규칙
- **변수명/함수명**: 영어(JavaScript, HTML, CSS 표준 준수)
- **클래스명/ID**: 영어(kebab-case 사용)
- **코드 주석**: 한국어 - 복잡한 로직이나 비자명한 부분만 주석 작성
- **예시**:
  ```javascript
  // 네비게이션 메뉴 토글 상태 추적
  let isMenuOpen = false;
  ```

### 문서 및 커밋 규칙
- **커밋 메시지**: 한국어 (예: "기능: 네비게이션 메뉴 구현")
- **문서화**: 한국어
- **파일명**: 영어 (파일명은 표준 준수)

## 코드 스타일

### HTML
- 의미있는 시맨틱 태그 사용 (header, nav, section, article, footer 등)
- 접근성을 고려한 ARIA 라벨 포함
- 적절한 heading 계층 구조 (h1, h2, h3...) 유지

### CSS / TailwindCSS
- TailwindCSS 유틸리티 클래스 우선 사용
- 커스텀 CSS는 필요한 경우에만 작성
- 2칸 스페이스 들여쓰기
- 색상, 폰트, 간격은 설계 토큰 일관성 유지

### JavaScript
- 화살표 함수 선호
- const/let 사용 (var 금지)
- 간단한 로직은 주석 생략
- 이벤트 리스너는 명확한 네이밍 사용

### 파일 생성 및 수정 시 주의사항
- HTML, CSS, JS 파일은 각각 분리하여 관리
- 스타일은 가능한 TailwindCSS 유틸리티 활용
- JavaScript는 바닐라 JS만 사용 (프레임워크 불가)

### 테스트 및 검증
- 브라우저에서 실제 동작 확인 필수
- 모바일(< 640px), 태블릿(640px-1024px), 데스크톱(> 1024px) 반응형 테스트
- 크로스 브라우저 호환성 확인

## 우선순위

1. **기능 정확성** - 이력서 콘텐츠 표시 및 상호작용 정상 동작
2. **반응형 디자인** - 모든 기기에서 잘 표시되어야 함
3. **성능** - 이미지 최적화, 로딩 속도
4. **접근성** - 스크린 리더 지원, 키보드 네비게이션

## 변경 사항 만들 때

- 한 번에 한 가지 기능만 구현
- 불필요한 리팩토링은 하지 않기
- 추측성 코드나 "나중에 필요할" 기능은 추가하지 않기
- 기존 코드와 스타일 일관성 유지
